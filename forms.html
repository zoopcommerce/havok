Forms

Dozens of reusable input widgets to prompt and validate all sorts of data.

Using Forms

Form widgets

Havok provides a wide range of enhanced form elements for the input and validation of data. They can be used stand alone. However, use inside a form provides powerful data binding functions.

Data binding

Example

Any havok form widges wrapped in a havok/form/Form will automatically data bind. Eg: 

<form data-dojo-type="havok/form/Form">
    <input data-dojo-type="havok/form/TextBox"
        name="something"
        data-dojo-props="
            label:'Something',
            helpMessages: 'please write something',
            value: 'example@example.com'
        "
     />
    <input data-dojo-type="havok/form/ValidationTextBox"
           name="somethingelse"
           data-dojo-props="label: 'Something Else', validator: 'Alpha'"
           value = "asdf123"
    />
    <input data-dojo-type="havok/form/DateTextBox"
        name="date"
        data-dojo-props="
           label: 'Date',
           helpMessages: 'please select a date'
        "
    />
</form>

Form value

The value of all the contained widgets can be set and retrieved through the form's value property. Use:

  • myform.set('value', valueObj) to set a form's value.
  • myform.get('value') to get a form's value.
  • myform.watch('value', function) to watch a form's value.

Form state

The state reflects the state of the contained widgets. If the state is an empty string, the state is good. If the state is something else, the form is not valid. Use:

  • myform.set('state', valueObj) to set a form's state.
  • myform.get('state') to get a form's state.
  • myform.watch('state', function) to watch a form's state.

Message position

There are three possible values for messagePosition:

  • auto: if the message is one line, display inline. If it is multiline, display block
  • inline: always display message inline. If the message is more than one line, only the first will be shown.
  • block: always display message as block, even when there is only one line.
<input data-dojo-type="havok/form/TextBox"
       data-dojo-props="
           label: 'name',
           messagePosition: 'block',
           helpMessages: 'Please enter your full name'
       "
/>

Post Activity

Textbox

Filters

Filters can be applied value displayed in the textbox. For more information about filters, see the filters section of the Data Quality documentation.

Note: The Trim filter is applied on all textboxes by default.

Example with an uppercase filter set:

<input data-dojo-type="havok/form/TextBox"
       value="John Smith"
       data-dojo-props="
           label: 'name',
           filter: 'Uppercase'
       "
/>

Set a filter chain:

<input data-dojo-type="havok/form/TextBox"
       value="    John Smith    "
       data-dojo-props="
           label: 'name',
           filter: ['Trim', 'Uppercase']
       "
/>

Filters are created by havok/filter/factory which supports abreviated names for the filters bundled with havok. To set your own custom filter, pass the complete module name. Eg filters: 'My/Filter/Module'

Validation Textbox

havok/form/ValidationTextbox extends havok/form/Textbox to add validation ability.

Example

<input data-dojo-type="havok/form/ValidationTextBox"
       data-dojo-props="validator: 'Alpha'"
       value = "asdf123"
/>

Specifying the Validator

Havok uses the mystique validator library. And, havok/validator/factory to create validator instances. The validator property will be passed to the validator factory to construct a validator.

Create a validator with constructor options:

<input data-dojo-type="havok/form/ValidationTextBox"
       data-dojo-props="validator: {base: 'Length', params: {min: 5, max: 10}}"
       value = "123"
/>

Create a validator chain:

<input data-dojo-type="havok/form/ValidationTextBox"
       data-dojo-props="validator: [
           'Alpha',
           {base: 'Length', params: {min: 5, max: 10}}
       ]"
       value = "123"
/>

Required and NotRequired Validators

If an input has a required attribute, a Required validator will automatically be added. If not, a NotRequired validator will automatically be added.

<input required data-dojo-type="havok/form/ValidationTextBox"/>

validationStyle

By default, validation errors are only highlighted after user interaction with the input. These styles be customised using the validationStyle property. It holds four lists of css class names that will be applied the four different validation situations.

<input data-dojo-type="havok/form/ValidationTextBox"
       data-dojo-props="
           validator: 'Alpha',
           validationStyle: {
               preActivity: {
                   valid: ['info'],
                   invalid: ['warning']
               },
               postActivity: {
                   valid: ['info'],
                   invalid: ['error']
               }
           }
       "
       value = "asdf123"
/>

postActivity

To force the postActivity style to be applied, set the postActivity flag to true.

<input data-dojo-type="havok/form/ValidationTextBox"
       data-dojo-props="
           validator: 'Alpha',
           postActivity: true
       "
       value = "asdf123"
/>

delay

To stop validation occuring on every single keystroke, there is a validation delay. It is set to 350ms by default. The example below has very slow validation with delay set to 2000ms:

<input data-dojo-type="havok/form/ValidationTextBox"
       data-dojo-props="
           validator: 'Alpha',
           delay: 2000
       "
       value = "asdf123"
/>

suppressValidation

The suppressValidation flag will stop all validation. It will not change the state of the textbox.

<input data-dojo-type="havok/form/ValidationTextBox"
       data-dojo-props="
           validator: 'Alpha',
           suppressValidation: true
       "
       value = "asdf123"
/>

suppressValidationMessages

The suppressValidationMessages flag will allow validation to continue, but will not show validation messages. Message suppression can be set for pre and post activity. Messages are suppressed pre activity by default.

<input data-dojo-type="havok/form/ValidationTextBox"
       data-dojo-props="
           validator: 'Alpha',
           suppressValidationMessages: {
               preActivity: false,
               postActivity: true
           }
       "
       value = "asdf123"
/>

Validation Group

havok/form/ValidationGroup provides the ability to do validation across multiple fields in a form. Place a ValidationGroup inside a form, and set a validator. Also set an array of field names in that form which will be passed to the validator. ValidationGroup will display any validation messages. Eg:

Example

<form data-dojo-type="havok/form/Form">
   <input
       data-dojo-type="havok/form/TextBox"
       value="Not Toby"
       data-dojo-props="label: 'username1', name: 'username1'"
   />
   <input
       data-dojo-type="havok/form/TextBox"
       value="Toby"
       data-dojo-props="label: 'username2', name: 'username2'"
   />
   <span
       data-dojo-type="havok/form/ValidationGroup"
       data-dojo-props="
       fields: ['username1', 'username2'],
       validator: 'havok/docs/module/MultiFieldValidator'"
   ></span>
</form>
